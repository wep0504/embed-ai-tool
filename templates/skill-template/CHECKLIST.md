# 新技能检查清单

- 选择一个小写连字符风格的目录名。
- 添加 frontmatter，并确保 `name` 与目录名一致，`description` 能准确描述触发场景。
- 使用 skill 模板中的全部必需章节。
- 复用共享约定，不要自行发明新的输出形状。
- 输出是否使用统一结构：`status`、`summary`、`evidence`、`next_actions`。
- 明确写出工具选择、配置选择和默认行为的优先级规则。
- 优先级是否与模板一致：`显式输入 > 工作区线索 > 历史上下文 > 默认值`。
- 写清这个 skill 可能输出哪些失败分类。
- 可恢复失败是否写明重试策略（次数、间隔、超限状态）。
- 是否定义了遇到歧义时的停止猜测条件（`ambiguous-context`）。
- 只有当平台差异会改变执行方式时，才写入平台相关说明。
- 至少说明一个现有 skill 作为下游交接目标。
- 是否写明“不交接直接结束”的条件。
- 执行 `python3 scripts/validate_repo.py`。
